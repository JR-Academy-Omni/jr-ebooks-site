开篇：一个人做 AI，最缺的从来不是想法

缺的是三样东西：撑过没收入那几个月的钱、跑模型的算力、还有一个能逼自己把东西真做出来的 deadline。这本书讲的，就是怎么在不融资、不贱卖股权的前提下，把这三样凑齐。

很多人卡在"我有个 AI 产品的想法，但没钱启动"。其实 2026 年对一个独立开发者来说，外面飘着的免费资源比任何时候都多——大厂为了抢开发者，把比赛奖金、API credits、加速器名额撒得到处都是。问题不是没有机会，是大多数人不知道这些机会存在，也不知道怎么够得着。

这本书不是一张机会清单（清单在最后两页的附录里）。它想跟你聊清楚三件事：这些钱对一个人意味着什么、动手前要准备什么、够得着它需要什么条件。机会信息都是 2026 年 6 月逐条查证过的，但比"有哪些机会"更重要的，是你看完知道自己该去够哪一个、怎么够。

黑客松 + 奖金赛。对一人公司，这是性价比最高的搞钱方式——见第 1、2 章。

02OPC 搞钱机会大全
第 1 章 · 打比赛这条路

为什么打比赛，是一个人最划算的搞钱方式

因为它一次性给你四样东西：现金、deadline、一个能写进简历的名次、还有大厂免费送的算力。找份兼职只给你钱，打一场比赛给你的远不止钱。

对有团队的公司，打比赛是锦上添花；对一个人，它几乎是最优解。你本来就在做一个 AI agent 或工具，比赛只是给你一个有截止日期的外部压力，逼你把"一直在做、永远做不完"的东西，收成一个"能交、能 demo"的版本。这个过程本身就是 OPC 反复强调的 MVP 训练。

先分清：黑客松 ≠ 奖金赛

这是两条不同的路，投错了白费力气。黑客松比的是"一个能 demo 的完整产品"，周期短（几天到两周）、看完整度和讲故事的能力，适合手里有 agent 产品、能快速 ship 的人。奖金赛（competition）比的是"一个明确指标上的分数"，周期长、看硬实力（模型/算法/榜单名次），适合技术底子扎实、愿意磨榜的人。

聪明的玩法是两条腿走：用同一套技术，把产品化的版本拿去打黑客松，把核心模型单独拎出来打奖金赛榜。一份功夫，两处收钱。

怎么判断你该打哪种
  • 你手里已经有能跑的 agent / 工具 → 打黑客松，几天就能交一版。
  • 你强在模型 / 算法 / 数据，但没做完整产品 → 打奖金赛，纯靠分数说话，不占股权。
  • 两样都有 → 两个一起报，同一套技术换两种壳。
  • 都还没有，纯新手 → 先打一场 MLH 这类新手友好的小赛练手，别一上来冲 $2M 大奖。

03OPC 搞钱机会大全
第 1 章 · 打比赛这条路

几个值得认真打的，说点掏心窝的话

AMD Developer Hackathon（$10K 现金 + AMD credits）——本期最推荐给大多数人。主题是 AI agent / workflow，正好是一人公司在练的产品形态；lablab.ai 社区竞争烈度中等，不像 YC 那样卷；最关键，AMD 的 API 兼容 OpenAI 格式，你现有的 agent 代码通常半天就能移植过去。投入小、命中率不低，先拿它练手最合适。

Build with Gemini XPRIZE（$2M，大奖 $500K）——别被金额吓到也别被它诱惑。它要的是"90 天内做出有真实用户和营收的 AI 业务"，本质是给成熟产品的，不是练手赛。手里已经有 MVP、有早期收入的，这是 8 月前最该认真打的一个；还在 0 到 1 的，看看就好，别 all in。

Built with Claude（$100K Claude credits）——agentic coding / Claude Skills 方向，有意思的是非程序员也赢过。如果你是用 AI 工具做产品的"半技术"创业者，这种赛反而对你友好，因为它奖励"想法 + 落地"而不是纯工程。

ARC Prize 2026（$2M 池，纯技术 solo 可打）——这是奖金赛里的硬骨头，追的是开源 AGI 推理。适合真正的技术派、愿意磨榜的人。它的里程碑奖（每档 $25K/$10K/$2.5K）比"通关大奖"现实得多，盯着里程碑打。

一个判断原则：先看竞争烈度，再看奖金。$2M 的赛全世界高手都来，$10K 的赛可能就几百人认真交。对一个人，赢一个小赛 > 在大赛里陪跑。

04OPC 搞钱机会大全
第 1 章 · 打比赛这条路

打一场比赛，怎么准备、需要什么

最大的误区是"等想好了再报名"。正确顺序反过来：先注册占位，拿到评审标准，再倒推你还差什么。报名都是免费的，占位不花钱，但能给你一个真实的 deadline。

第二个误区是"为每个比赛从零做"。别。前面那些比赛的内核，对你来说都是同一个东西——一个能跑的 AI agent / 工具。手里养一个主产品，按每个比赛的要求换壳：AMD 赛把模型调用换成 AMD Cloud，Gemini 赛接一个 Google Cloud 服务，多 agent 赛把单 agent 拆成协调版。同一套核心逻辑，投五六个赛，命中率自然高。

如何准备 · 三步
  • ① 今晚就注册占位：选 1 个主赛（还在练→AMD；有用户有收入→Gemini XPRIZE），免费报名，拿到评审标准。
  • ② 倒推差距：照评审标准列出"我还差什么"，砍掉一切不影响评分的功能。
  • ③ 用现有代码改一版：不要新开项目，先交一版能跑的，再迭代。
需要什么 · 硬条件
  • 一个能跑通的 demo：跑不起来直接出局，提交前在干净环境复测。
  • 规定的技术栈：大厂赛要求"必须用 X 产品"，没用就废。
  • 三件套齐全：开源链接 + 60 秒 demo 视频 + 报名表单，缺一不收。
  • 资格符合：solo 是否允许、人数上限、是否要求比赛期间新建。
05OPC 搞钱机会大全
第 2 章 · 怎么交出能拿奖的东西

48 小时，怎么做出评审认的 demo

一个人打比赛，时间是唯一的稀缺资源。能不能拿奖，往往不取决于你技术多强，而取决于你有没有把有限的时间花在评审真正看的地方。

scope 怎么砍：把产品砍到一个"输入 → AI 处理 → 输出"的闭环就停。别做登录注册、别做后台管理、别追功能完整度。评审看的是"这个 agent 真能干一件有用的事"，不是"你做了多少功能"。一个能跑通的小闭环 + 一段讲得清楚的 demo 视频，胜过十个半成品功能。

评审实际打分的，就这四件事

① 真能跑——评审时间紧，跑不起来等于零分。② 解决真问题——能一句话说清"谁的什么痛"，比技术炫技得分高。③ 真用了赞助方的栈——大厂赛的隐藏评分项，别只挂个名。④ demo 讲得清——60 秒视频 + 一句话定位，评审第一眼就懂你做了啥。

最常见的翻车 · 别踩
  • 超期 / 漏交材料：开源链接、demo 视频、表单三样缺一，直接不收。
  • 复用旧项目：多数赛要求比赛期间新建，翻 commit 历史能查到，会被 DQ。
  • 没满足赞助方硬要求："必须用 X 产品"是硬条件，没用就废。
  • 资格超限：报名前看清 solo 是否允许、团队人数上限。
06OPC 搞钱机会大全
第 3 章 · 不打比赛也能拿钱

加速器、Grant、免费 credits，先分清本质

这三样经常被混在一起说"融资渠道"，其实代价完全不同。搞错了，你可能为了几万块算力把早期股权贱卖了。

加速器（YC / Antler / Techstars）= 用股权换钱 + 网络 + 背书。它给你一笔投资（$15 万到 $50 万不等），但要走你 5%~12% 的股权。值不值，看你缺的是不是"钱之外的东西"——SF 的人脉、投资人对接、品牌背书。如果你只缺启动资金，为了"进个名营"把股权给出去，不划算。

政府 / 机构 Grant（CSIRO / SBIR / EIC）= 给钱，不要股权。但天下没白拿的钱：它们要求当地注册、有时要你出等额配套现金、而且大多是给你"解一个有技术不确定性的研究问题"，不是给你做产品。申请周期长、材料硬，但拿到了一分股权不稀释。

免费 API credits（云厂 + 模型厂）= 不是钱，是省钱，但启动期最实用。有几家常年可申、不要股权、门槛低（Microsoft Founders Hub、AWS Activate、Hugging Face 这类），省下的算力钱够你跑好几个月。但要小心：网上把所有大厂 credits 都说成"随便申"，实际 OpenAI、Anthropic 的 credits 都卡 VC / 加速器背书，纯自筹申不到——下一页给你分清哪些真能薅、哪些是坑。

怎么选 · 一句话
  • 缺钱 + 缺网络背书 → 考虑加速器（愿意给股权）。
  • 缺钱、但不想稀释 + 能满足资格 → 申政府 Grant。
  • 只是想省算力 / API 成本 → 先把免费 credits 全申一圈，这个零成本、零风险。
07OPC 搞钱机会大全
第 3 章 · 不打比赛也能拿钱

Grant 怎么申不白费力 · credits 怎么薅一圈

Grant 申请最容易白费力的地方，是花两周写完材料才发现自己根本不符合资格。所以顺序一定是先查硬条件，再决定写不写。

Grant 申请三个要点

① 先看资格硬条件：注册地、公司年龄、营收上限、是否要配套现金——不符合直接跳过。② 对齐它的"使命"：政府 grant 都有政策目标（深科技 / 社会影响 / 本地就业），你的提案要往它的话术上靠。③ R&D 类要有"研究问题"：CSIRO、SBIR 不是给你做产品，是给你解一个有技术不确定性的问题，提案里要写清"难在哪"。

实测下来，一个人能自助申到、不换股权的就这几个
  • Hugging Face ZeroGPU：注册免费账号就送每日免费 GPU 额度，门槛全场最低，跑 Gradio demo 直接用。
  • Microsoft for Startups Founders Hub：三大云里对一人公司最友好。自助申、不要投资人、不换股权，按用量渐进解锁，最高 $150K Azure（含 OpenAI 访问）。
  • AWS Activate（Founders 档）：自筹也能直接申，$1K–$5K，不要加速器、不换股权。
  • Google Cloud（Start 档）：自助可申 $2K，无需投资人。
  • ElevenLabs Grants：员工 <25 人即可，3300 万字符 / 12 个月，代价是网站底部挂它 logo。
  • NVIDIA Inception：免费加入、不换股权，但价值是硬件折扣 + 合作云通道，不是白送现金，别冲着免费 GPU 去。

这几个别白费力 · 完整操作包在创业营里

这本册子敢把"哪些其实申不到"也写出来，是因为画饼最害人——你照网上的攻略熬两周写材料，最后发现压根不符合资格。先说实话，再说哪里能拿。

网上传得火、一个人其实申不到的（实测）
  • OpenAI for Startups：没有给无背书 indie 的自助通道，必须 VC / 加速器背书——最大的坑。

  • Google $350K / AWS $200K 大额档、CSIRO（要澳洲注册+自掏配套现金）、EIC（须在欧洲设立）、美国 SBIR（要≥51% 美籍持股）：门槛都在那，海外一人公司基本无缘。

官方链接附录里都给了，"怎么申得上"在营里

09OPC 搞钱机会大全
第 4 章 · 这些钱在一人公司里怎么用

搞到钱之后，别把顺序搞反

奖金、grant、credits 凑起来，能让你扛过没收入的早期，不用一上来就去融资贱卖股权。但要记住：这些是燃料，不是目的地。

见过太多人沉迷"打比赛"本身——这周一个黑客松，下周一个奖金赛，奖金是拿了不少，产品却始终停在 demo 阶段。这就是把顺序搞反了。对一人公司，正确的用法是：

💵

奖金 = 第一桶启动钱

凑起来够你扛过没收入的早期，不用先融资。但它是阶段性的，不是稳定现金流。

🏅

名次 = 硬背书

获奖 / 入围是客观信号，接单、融资、招人时都能拿出来说，比自我介绍有用。

⏱

deadline = 逼出 MVP

外部压力逼你把"在做的"收成"能交的"，这正是产品从 0 到 1 最难的一步。

🎯

但主线永远是产品

真正的盘是把比赛产品接着做成有付费用户的东西。别为打榜耽误做产品。
