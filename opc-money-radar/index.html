奖金赛 vs 黑客松：先分清再投

奖金赛比"一个指标的成绩"，黑客松比"一个能 demo 的产品"。奖金赛周期长、看硬实力（模型 / 算法分数），适合技术底子扎实、愿意磨榜的人；黑客松周期短、看完整度和讲故事，适合手里有 agent 产品的人。两类可并行：用同一套技术，一边打榜拿奖金、一边包成产品打黑客松。

① 先看资格硬条件：注册地、公司年龄、营收上限、是否要配套现金——不符合直接跳过，别浪费两周写材料。② 对齐它的"使命"：政府 / 机构 grant 都有政策目标（深科技 / 社会影响 / 本地就业），你的项目要往它的话术上靠。③ R&D 类要有"研究问题"：CSIRO / SBIR 不是给你做产品，是给你解一个有技术不确定性的问题，提案里要写清楚"难在哪"。

09OPC 搞钱机会大全

🎯 实操一：一个产品，打多个赛

别为每个比赛从零做。

上面几十个机会的内核，对一人公司来说都是同一个东西：一个能跑的 AI agent / 工具。聪明的做法是手里养一个主产品，按每个比赛的要求换壳——AMD 赛把模型调用换成 AMD Cloud；Gemini XPRIZE 接一个 Google Cloud 服务；Built with Claude 改成 Claude Skills 版；多 agent 赛把单 agent 拆成协调版；奖金赛就把核心模型单独拎出来打榜。同一套核心逻辑，投五六个赛，命中率自然高。

🚀

这周就能做的三步

① 选一个主赛：还在练 → AMD（现金、竞争中等、半天能改）；有用户有收入 → Gemini XPRIZE。② 今晚免费注册占位，拿到评审标准，倒推还差什么。③ 用现有代码改一版，先交一版能跑的再迭代，别新开项目。

⏱

48 小时 demo 怎么 scope

砍到一个"输入→AI 处理→输出"的闭环就停。别做登录注册、别做后台、别追完整度。评审看的是"这个 agent 真能干一件有用的事"，不是功能多。录一段 60 秒能跑通的 demo 视频比什么都重要。

10OPC 搞钱机会大全

🎯 实操二：评审看什么 · 常见翻车

评审实际打分的四件事

① 真能跑：评审时间紧，跑不起来 = 直接出局。提交前在干净环境复测一遍。

② 解决真问题：能一句话说清"谁的什么痛"，比技术炫技得分高。

③ 用对了赞助方的栈：大厂赛的隐藏评分项就是有没有真用它家产品，别只挂个名。

④ demo 讲得清：60 秒视频 + 一句话定位，评审第一眼就懂。

最常见的几个被 DQ / 扣分点

· 超期 / 漏交材料：很多赛要求开源链接、demo 视频、表单三样齐全，缺一不收。

· 复用旧项目：多数赛要求比赛期间新建，翻 commit 历史能查到。

· 没满足赞助方要求："必须用 X 产品"是硬条件，没用就废。

· 团队人数 / 资格超限：报名前看清 solo 是否允许、人数上限。
