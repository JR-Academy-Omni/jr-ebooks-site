03

一个产品，打多个比赛

怎么排
别为每个比赛从零做。这 5 个比赛的内核都是"一个能跑的 AI agent / 工具"。聪明的做法是：手里养一个主产品，按每个比赛的要求换壳 —— AMD 赛就把模型调用换成 AMD Cloud，Gemini XPRIZE 就接一个 Google Cloud 服务，Band of Agents 就把单 agent 拆成多 agent 协调。同一套核心逻辑，投三四个比赛，命中率自然高。
1

这周就能做的三步

  • 选一个主赛：手里产品还在练 → 选 AMD（现金、竞争中等、半天能改）；已经有真实用户和收入 → 直接备 Gemini XPRIZE。
  • 今晚注册占位：报名免费，先注册拿到提交入口和评审标准，按标准倒推还差什么。
  • 用现有代码改一版：不要新开项目。把你正在做的 agent 改成符合该赛要求的版本，先交一版能跑的，再迭代。
2

打比赛在 OPC 里是什么位置

  • 验证：比赛是一个有 deadline 的外部压力，逼你把"在做的东西"收成"能交的东西" —— 对应 OPC 的 MVP 模块。
  • 曝光与背书：获奖 / 入围本身是简历和融资材料里的硬信号，比自说自话有用。
  • 不是终点：奖金是第一桶启动钱，真正的盘是把比赛产品接着做成有付费用户的产品 —— 这才是一人公司的主线。
